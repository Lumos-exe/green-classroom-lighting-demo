# Agent 项目任务书：基于真实教室 3D 模型的智能节能灯光控制演示视频与研究图表

## 0. 当前项目状态

本项目已经完成并保留了一个较满意的真实教室 Blender 静态模型。后续任务不是重新建模，而是在此模型基础上继续开发：

```text
blender/create_realistic_classroom_preview.py
outputs/blender/showcase_classroom_scene.blend
outputs/videos/classroom_model_rear_to_front.png
outputs/videos/classroom_model_lighting_low.png
outputs/videos/classroom_model_lighting_compare.png
true_classroom_images/
CURRENT_3D_MODEL.md
```

必须严格复用当前教室模型的空间结构、座位布局、阶梯、窗户、门、讲台、黑板、灯具、材质、相机方向和比例关系。不要推倒重建教室，不要退回粗糙示意模型。

当前模型的重要基准：

- 小教室尺度，约 12 排固定连排桌椅。
- 前半教室为平地，后半教室为 6 个等距阶梯。
- 排距和台阶踏面深度绑定，当前为约 `0.90m`。
- 地面有 0.6m 砖缝网格，阶梯砖缝按“前面整砖、后面半砖”处理。
- 窗户在后往前视角的左侧，门在右侧长墙。
- 默认主相机为后往前看讲台，贴近后墙，能看到讲台、黑板、座位、窗户、门和 5 行顶灯。
- 顶灯为 5 行 × 4 列，共 20 个独立灯具；前方讲台/黑板区域有 2 条独立线状灯。
- 灯光必须通过真实 Blender Area Light 能量变化照亮场景，不能靠改变曝光、整体环境光或材质变白来假装开灯。

## 1. 总目标

在现有 3D 教室模型基础上，制作一个用于“绿色节能、人员活动感知、智能照明调控”的展示成果，包括：

1. 一个约 20 秒的 Blender 演示视频；
2. 一个或多个灯光亮度变化辅助可视化，例如嵌入视频的小型实时亮度示意图、单独 GIF、或配套视频；
3. 一组用于课程报告/展示的研究图表，包括但不限于能耗对比、人员密度热力图、分区照明曲线、模式切换时间线等。

本项目不是追求真实工程部署，而是做一个逻辑可信、视觉清晰、可解释性强、适合课程汇报的演示系统。

## 2. 最重要原则

### 2.1 以当前 3D 模型为准

后续所有脚本必须基于当前 `create_realistic_classroom_preview.py` 或其复制/扩展版本开发。允许新增动画脚本，例如：

```text
blender/create_smart_lighting_demo.py
```

但不得删除或破坏当前静态模型脚本中的真实教室结构。新增脚本应尽量复用当前模型创建函数，避免复制后结构漂移。

### 2.2 灯光变化必须真实可见

灯光控制必须改变独立灯具的 Area Light 能量。禁止用以下方式冒充灯光控制：

```text
改变相机曝光
整体提高世界环境光
把墙面/桌面/地面材质改白
全局加大面积补光掩盖分区差异
直接把画面后期调亮
```

灯罩自身可以使用低强度可见发光材质，目的是让灯具看起来亮；但真正照亮桌面、地面、墙面、座椅的必须是对应 Area Light。

### 2.3 低亮不等于全黑

无人/安全模式应明显偏暗，但仍可看清教室轮廓、座位、讲台和主要结构。演示视频需要观众理解空间，不应让画面黑到无法辨认。

### 2.4 自习模式是重点

自习模式必须清楚表现“有人区域亮、无人区域暗”。不是简单整间教室降低亮度，也不是有人的地方纯白、没人的地方全黑，而是模拟真实灯具分区控制：

- 有人的座位附近灯具提高亮度；
- 相邻区域有一定光照过渡；
- 无人区保持低亮或安全亮度；
- 桌面、地面和墙面应能看出局部明暗差异。

## 3. 20 秒视频内容规划

视频总长约 20 秒，建议 24fps，约 480 帧。可以根据渲染速度略微调整，但总时长应接近 20 秒。

推荐时间线：

```text
0.0s - 2.5s    无人安全模式
2.5s - 6.5s    上课入场与上课模式
6.5s - 9.5s    课间模式
9.5s - 13.0s   投影模式
13.0s - 18.0s  下课离场与自习模式
18.0s - 20.0s  总结镜头 / 稳定展示最终自习分区照明
```

可以细调，但必须包含以下模式，并且模式切换要有平滑过渡，不要闪烁：

### 3.1 无人安全模式

- 教室几乎无人。
- 所有灯具处于低亮安全照明。
- 画面偏暗但可读。

### 3.2 上课模式

- 学生陆续进入并坐下。
- 灯光从安全亮度逐步升到上课亮度。
- 座位区、讲台区、黑板区都逐渐变亮。
- 重点让观众看到“有人进入 -> 灯光响应”的关系。

### 3.3 课间模式

- 部分学生离开座位，到过道或门口附近活动。
- 过道、门口、前后通行区域亮度提高。
- 座位区可适度降低，但不能完全黑。
- 人物活动不能穿过桌椅，必须沿中间过道、侧边过道和排间可通行区域。

### 3.4 投影模式

- 老师放投影或屏幕展示。
- 黑板/前排强照明降低，避免投影区域过亮。
- 讲台和必要通行区域保留安全亮度。
- 屏幕/投影区域成为视觉中心。

### 3.5 自习模式

- 下课后大部分学生离开。
- 只剩零散几个人分布在不同座位。
- 有人附近灯具保持较高亮度，无人区域降到低亮。
- 要明确体现节能调光的价值，是本视频最重要的展示片段之一。

## 4. 人物与路径要求

人物不追求高精度动作捕捉，但必须可信，不要滑稽。

### 4.1 人物模型

- 使用简洁低多边形完整身体，不要只有球形头。
- 区分站立、行走、坐下三种高度和姿态。
- 颜色可适度区分学生和教师。
- 人数不需要很多，但要足以表现模式变化。建议：
  - 上课模式：约 20-35 人，可用低多边形角色。
  - 课间模式：约 8-12 人在过道/门口活动。
  - 自习模式：约 4-8 人零散分布。

### 4.2 路径约束

人物必须沿真实路径移动：

```text
门口 -> 侧边过道 / 中间过道 -> 排入口 -> 座位
座位 -> 排入口 -> 过道 -> 门口
```

禁止：

```text
穿过桌椅
穿过墙
穿过讲台
直接直线穿越座位块
悬空或陷入地面
```

如果动画时间有限，可以使用简化运动，但路径必须合理。人物进入座位时，应先到排入口，再沿排内短距离移动并坐下。

## 5. 逐灯具精细控制要求

当前模型有 20 个顶灯，命名应保持可控，例如：

```text
ceiling_light_r01_c01 ... ceiling_light_r05_c04
front_linear_light_left
front_linear_light_right
```

必须按“单个灯具”细致调节。20 个顶灯和 2 条前方线灯都必须是独立控制对象，不能只做粗糙的一块块整体调光。

禁止以下实现方式：

```text
把前区/中区/后区整块灯具设成同一个亮度
把左/中/右整列灯具设成同一个亮度
只给所有灯具一个全局亮度倍率
自习模式只把整片座位区统一变亮或变暗
```

区域概念只能用于计算目标亮度或生成图表，不能替代逐灯具控制。最终动画和数据中，每盏灯必须有自己的亮度值、关键帧和时间序列。

可以用以下方式计算单灯目标亮度：

- 根据每盏灯到人员/座位/过道活动点的距离衰减；
- 根据当前模式叠加前区、讲台、投影、过道等需求权重；
- 对相邻灯具做适度平滑，避免亮度突变；
- 对无人区保留低亮安全照明，而不是直接归零；
- 对投影模式降低靠近投影/黑板前区的灯具亮度。

区域划分可以作为辅助参考，例如：

```text
前 / 中 / 后 × 左 / 中 / 右
```

但最终必须落到每个灯具行列坐标：

```text
row 1-5
col 1-4
```

每个灯具应有：

- Area Light 能量；
- 可见灯罩亮度；
- 当前模式下的目标亮度；
- 平滑过渡关键帧。
- 在 `outputs/data/light_brightness_timeseries.csv` 中的独立列或独立记录。

建议输出的数据结构至少包含：

```text
time_s
mode
ceiling_light_r01_c01
ceiling_light_r01_c02
...
ceiling_light_r05_c04
front_linear_light_left
front_linear_light_right
```

所有亮度变化必须插值平滑。可以使用 `ease in/out` 或逐帧计算后插入关键帧。

## 6. 视频画面与辅助可视化

默认视频应使用后往前主视角，因为它能同时看到讲台、黑板、座位、窗户、门和灯具。

如果后方视角中局部灯光变化不够明显，必须增加辅助可视化，至少一种：

1. 在视频角落嵌入实时灯光亮度矩阵；
2. 生成单独 GIF 显示 5×4 顶灯亮度变化；
3. 生成配套 MP4，展示顶视角灯具亮度和人员占用变化；
4. 生成每个模式的静态对比图。

辅助图应和视频时间线匹配，例如显示：

```text
当前时间
当前模式
20 个顶灯亮度矩阵
前方两条线灯亮度
人员占用热力
预计相对能耗
```

## 7. 研究图表要求

### 7.2 灯光亮度曲线

```text

```

### 7.3 5×4 顶灯亮度热力图

```text

投影模式
自习模式
```

每张图显示 20 个顶灯亮度。自习模式必须显示明显局部高亮。

### 7.4 人员密度/占用热力图

用教室平面坐标生成热力图，显示不同模式下人员分布。至少包含：

```text
上课模式人员密度
课间模式人员密度
自习模式人员密度
```

### 7.5 能耗对比柱状图

对比至少三种策略：

```text

智能分区调光
```

输出相对能耗、节能率。数值可以基于灯具亮度积分估算，不需要真实电表数据，但计算逻辑必须清楚。

### 7.6 照明满足率 / 过照明比例

可用简化评价：

- 有人区域亮度是否达到目标；
- 无人区域是否被过度照亮；
- 投影模式前区是否过亮。

可以输出折线图、柱状图或雷达图。

## 12. 验收标准

最终成果至少满足：

- 当前真实教室模型结构没有被破坏。
- 视频约 20 秒，包含无人、上课、课间、投影、自习五类模式。
- 灯光变化是通过独立 Area Light 能量控制实现。
- 自习模式能清楚看到有人区域亮、无人区域暗，但无人区不是纯黑。
- 人物不穿桌椅、不穿墙、不悬空。
- 后往前视角能看到整体教室、讲台、黑板、座位和多行顶灯。
- 若视频中灯光变化不够直观，必须提供同步亮度矩阵 GIF/视频/嵌入图。
- 输出至少 5 类研究图表，并保存对应数据。
- 所有脚本可在容器内运行。
